Making a Basic GET Request

Now that we've got Postman set up, we can make our first call! The first one we're going to make is to the Pokemon API, which documentation can be found here: PokeApi. We're starting with this one because they don't require any authorization, and because Pokemon! It also has a great interface and decent documentation for getting started. (If you don't know anything about Pokemon, they're basically cute characters from a video game that can be caught and then used to battle; each Pokemon has different attacking abilities and strength, and some of them can even evolve into other Pokemon.)

To get started, make a new request in Postman, if one isn't open already:

Then in main bar, type the URL as shown on the main page of the PokeApi website, like so:

  1. Request Method - here is where you can specify the request method (GET, POST, PUT, etc.)
  2. Request URL - the actual route for the API, including any required variables
  3. Send - actually make the request
  4. Response Body - the response from our request; notice Postman also records the status (200 OK in the example above), how long the request took, and how big the response is

Go ahead and try entering a few different Pokemon. What happens if you enter a name that doesn't correspond to an existing Pokemon? (Check the status!)

Now let's practice using the docs. Click on the "Check out the docs!" link on the main page or go to this URL: https://pokeapi.co/docs/v2.html. Not all documentation is equal, and as you enter the development world, you'll begin to appreciate good documentation. After the initial introductory and legal information, the left navigation provides a breakdown of the various resources that are available in this API. For example, if we click on "Berries", it shows us that, in order to retrieve berry information, we'll use the GET method on the endpoint "https://pokeapi.co/api/v2/berry/{id or name}".

Go ahead and peruse the documentation and try several different requests!
