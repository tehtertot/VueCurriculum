Cloud Firestore

Objectives:

  • Understand what Cloud Firestore is
  • Learn how to set up a Cloud Firestore database

Cloud Firestore is Firebase's NoSQL database. If you've worked with databases using SQL, then you've worked with a relational database. This is where data is stored in tables with columns of data, and those tables have keys that allow the tables to be joined together. In contrast, a NoSQL or non-relational database stores data in documents, which are held in collections. When working with databases, there are some key operations we'll typically use to interact with them:

  • Create - add new data
  • Read - retrieve/view some or all of the data
  • Update - edit existing data
  • Delete - remove existing data

Setting up a New Cloud Firestore Database

If we want to use the database, we'll first have to set it up in our Firebase console. We've already created our first project. From there:

  1. Either click on the Cloud Firestore card in the main window under "Choose a product to add to your app" or click on the "Database" option in the left Develop menu.
  2. Click "Create database"
  3. Since we're just learning, let's select test mode to start
  4. The next screen prompts you to select a location. Since our projects will be relatively light and small, we don't have to worry about location too much, so you can go ahead and select the default. (In the real world, though, you may want to consider choosing a location based on where you anticipate the majority of your users to be located, as physical proximity can result in improved performance.)
  5. Click "Done." It may take a minute or two to set up, but that's because Google is managing several configurations so we don't have to worry about it on our end!
  6. To get used to the notion of collections and documents, go ahead and add a collection and a few documents to that collection.
