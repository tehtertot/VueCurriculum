Authentication

Objectives:

  • Learn why authentication is important
  • Learn how to authenticate with Firebase

As we develop web applications for users, authentication is a very important process to consider. Authentication is the process of verifying that someone is who they say they are. So far, none of our applications have utilized this process--anyone could walk up and play Ninja Gold or browse through the applications we've made; but suppose we did want it to matter. What if we wanted to keep track of Ninja Gold high scores or a history of game play per user? You have probably used dozens of sites that require you to authenticate by signing in with some combination of username and password or log in via an existing account like Google, Facebook, or GitHub.

While this is certainly something we could manage internally in our application (by saving usernames and encrypted passwords into a database and then checking that database each time a user logs in), we are going to utilize a third-party API, namely Firebase, to assist us with this process. We can trust a provider like Google to ensure our data is secure, and that way, we don't have to be experts in security; instead, we can focus on the heart of our application!

Setting Up Firebase Authentication

This assumes you have already set up your Firebase account at the end of the last module. If you haven't, be sure to do that now (https://firebase.google.com/)

  1. Create a new Vue application
  2. Sign in to Firebase and click on "Go to console" in the upper right corner.
  3. Click on the "Create a project" button in the middle of the screen (or "+ Add a project" if you have any existing projects).
  4. Walk through each of the next steps:
    1. Give your project a name. This will be be how you reference it later.
    2. Leave the default "Not right now." (If you already have a Google Analytics account set up, feel free to leave this on. We just won't be exploring it as part of this track.)
  5. Once your project is ready, click "Continue."
  6. On the left navigation bar, under "Develop," click "Authentication."
  7. In the main pane, select the "Sign-in method" tab.
  8. Under "Provider," click on Email/Password to edit it. Click the "Enable" toggle to turn this option on. (For now, leave the Email link (passwordless sign-in toggled off).
