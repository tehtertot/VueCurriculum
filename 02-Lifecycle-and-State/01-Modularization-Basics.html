Modularization Basics

Objectives:

  • Understand why modularization is important.
  • Understand the basics of modularizing vue components.

A quick overview

Modularization is an important concept for building Vue applications. In fact, it's what helps make Vue so powerful. Before we dive into the details, let's take a step back and talk about modularization in general.

Modularization is concept of breaking our code into separate files, or modules, so that our code becomes more manageable and maintainable. There's actually nothing Vue specific about modularization. Modulariztaion is widely used in programming in general to help manage codebases of all shapes and sizes. In fact, we've already used modularization in the past. Remember when we took our CSS and put it into a file separate from our HTML files? That's modularization!

How does modularization make our code more maintainable?

Let's imagine a scenario where our app has started to gain traction, but we need to add more features to support our growing user base. After a little while, our App.vue file has gone from 100 lines of code to 5000 lines of code. Now, when we want to find a component, it might take minutes of searching before we can locate where it was defined. Once we find it, we may need to cross-reference it with another component, so we have to search for minutes once again. It's easy to see how this is unmanageable.

Enter modularization. We'll split all our components into separate files. Each component will be exported from the file where it's defined and imported into each file that relies on it. That way, it's easy to find everything just by looking at our folder hierarchy.

Modularization in Vue

When we create a new Vue app using the Vue CLI, we can see that Vue is already beginning to start the modularization process for us. If we run npm run serve, we'll see that there's plenty of content available to our browser, but where is it all coming from? Let's start by taking a look at the App.vue file.

src/App.vue

Notice that the html template in this file is rendering the HelloWorld component. We don't just magically get access to an html tag called HelloWorld, check out the javascript between the script tags. Notice that we're importing HelloWorld from a file in the components folder. Now take a look at the export. App.vue is a component itself, and it is registering HelloWorld as one of its child components.

Moving forward we'll need to remember all three of these steps when we want to start modularizing our component structure.

  1. Import the component you'd like to use in your file.
  2. Register the component under the `components` key.
  3. Make sure the component you're trying to import is being correctly exported from its file.
