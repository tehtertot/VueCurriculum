Mapping Vuex Mutations

Objectives:

  • Learn a more concise syntax for mutating state

If you look at our last lesson, you'll see a lot of code, and some of it is definitely unnecessary. Let's pick off a low-hanging fruit and modify our MovieListItem component to use the mapMutations function.

Before:

  export default {
    name: "MovieListItem",
    props: {
      movie: {
        id: Number,
        score: Number,
        title: String,
        genre: String
      }
    },
    methods: {
      onUpvote() {
        this.$store.commit("incrementMovieScore", this.movie.id);
      },
      onDownvote() {
        this.$store.commit("decrementMovieScore", this.movie.id);
      }
    }
  };

After--html:

  <template>
    <div class="movie">
      <h5>{{ movie.title }}</h5>
      <span class="score">score: {{ movie.score }}</span>
      <div class="voting-buttons-container">
        <button @click="incrementMovieScore(movie.id)">Upvote</button>
        <button @click="decrementMovieScore(movie.id)">Downvote</button>
      </div>
    </div>
  </template>

After--script:

  import { mapMutations } from "vuex";
  
  export default {
    name: "MovieListItem",
    props: {
      movie: {
        id: Number,
        score: Number,
        title: String,
        genre: String
      }
    },
    methods: {
      ...mapMutations(["incrementMovieScore", "decrementMovieScore"])
    }
  };

Look at how much more concise our methods property is. And all we had to do was use the mapGetters helper and update the html click handlers. Don't forget to use mapMutations when you can!
