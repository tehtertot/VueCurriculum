Intro to Vuex

Objectives:

  • Understand the purpose of the Vuex library
  • Understand when to use Vuex

What the heck is Vuex?

We've alluded to Vuex before when we were talking about state management, and now it's finally time to dive in. Vuex is simply a state management library that allows us to maintain a global store of information. Just like we did on our own, Vuex will create a single state object, and we can access that state from any component. We'll also only ever modify state indirectly by calling a function from our components. There may be some new concepts in this chapter that feel overwhelming, so if you start to feel overwhelmed, just come back to this section and remind yourself what Vuex is all about.

So what's different about Vuex and why do we care?

When we use Vuex, it's going to feel very similar to the store pattern that we used earlier. Eerily similar in fact. That's because it is! So why do we need it? Vuex does two major things for us:

  1. Provides conventions for how we structure and modify our data using mutations, actions, and modules.
  2. Gives us debugging tools "out of the box" that allow us to track how our application's data changes over time.

Vuex also gives us a few additional helpful tools and "nice-to-haves", but the two reasons above are the meat of the library's usefulness.

When should we use Vuex?

Vuex is really useful for debugging and structuring applications that we expect to get reasonably complex. Typically, you'll want Vuex if you know your project is going to be medium or large in size or complexity. Well that's great to know, but how do we know when our app is medium or large? The thing is, you kind of just know when you know. But here are some things to look for:

  • You find yourself struggling to debug how data is changing in your application.
  • You find yourself wishing that you could break your store up into separate parts.
  • You just want the structure because you know you want your app to grow to be pretty large.

Any of the above are good reasons to switch, but you might find that sometimes you're fine with your own custom store. Either way, if you're handling data using your own store pattern, it should be really easy to switch to Vuex when you need to, so rest easy and feel it out!
