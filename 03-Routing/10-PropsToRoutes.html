Passing Props to Route Components

Objectives:

  • Learn how to move route parameters to props
  • Understand why this is desirable

It is a little cumbersome to always refer to the route via $route.params. From a programmming standpoint, it is also tightly coupled, which, as a rule, is something we try to avoid where possible. To be tightly coupled means that this component will only be available when the route specifies a value and directs to the component. By moving the value to the component's props, we now have the flexibilty of using this component as described above, or using it by passing a prop value to it via other means.

VueRouter actually abstracts much of this away from us, so the process is quite simple. In the route, we just need to set the props option to true:

src/router/index.js

    {
        path: '/user/:id',
        component: User,
        name: 'user',
        props: true         // added this line
    }

Now, in the component where we previously had something like this in our template -- {{ $route.params.id }} -- we can specify the props using the variable name we specified in our path definition:

src/components/User.vue

    <script>
        export default {
            name: 'User',
            props: {
                id: Number
            }
        }
    </script>

This means we can now reference the variable directly in our template:

src/components/User.vue

    <template>
        <p>Viewing user #{{ id }}</p>
    </template>
