Nested Routes

Objectives:

  • Understand how to coordinate nested components with routes

Just as components can be nested, routes can also be nested. While we don't necessarily need a route for every single nested component, designing with nested components and associated routes can help us design and organize our application in an intuitive and user-friendly way. Think back to the last assignment and consider the single restaurant page. Perhaps we want to have a component for the whole restaurant, but within that component want to switch between photos, reviews, and other information. Nested components make this relatively easy, and adding routing helps us keep track of what our user is doing and can provide a better navigated experience.

The last piece to remember is that for any component with child routes, the parent component should have a router-view component in its template wherever the child component should appear. For the example above, it might be something like this:

src/components/HomeComponent.vue

    <div>
        <p>This is in the parent component</p>
        <router-view></router-view>
    </div>
