Named Routes

Objectives:

  • Learn about named routes
  • Learn how to pass parameters with named routes

Sometimes string interpolation or concatenation can get a bit messy, as path names can get rather lengthy. For convenience in routing, there is an additional option, name, we can add when declaring our routes:

src/router/index.js

    {
        path: '',
        component: Home,
        name: 'home',   // added this; does not have to match the component name, but it sure helps!
    }

Then, in our RouterLink component, rather than providing the literal path, we can specify the name:

    <router-link v-bind:to="{ name: 'home' }">Home</router-link>

If we've got any additional parameters, we can pass an object specifying those value(s). Suppose, for example, we had a route like so:

src/router/index.js

    {
        path: '/user/:id',
        component: User,
        name: 'user',
    }

We would pass the value for id through the params key as an object, where the object's keys correspond to the variable names specified in the path.

    <router-link v-bind:to="{ name: 'user', params: { id: '42' } }">Link to Specific User Page</router-link>
