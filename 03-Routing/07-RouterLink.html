Router Link

Objectives:

  • Learn how to add href links that use our app's routes
  • Learn about the router-link component

Up to this point, we've had to manually type in the URL to navigate to different views. This is certainly not the ideal user experience, so VueRouter has a RouterLink component that allows us to add internal links. Of course, if needed, the regular <a> tag is available, but should be reserved for external links; but the RouterLink component is built to intercept the link to ensure the browser doesn't reload the page (which is important in a SPA). When we use the RouterLink component, we don't actually have to create the <a> tag--it will be created for us. When we implement this, be sure to inspect the element to see what actually gets rendered.

To use the RouterLink component, we just need to put the clickable text between the opening and closing tags (just as we would with an anchor tag), and then use the v-bind:to prop to specify which route this link should be tied to. When specifying the path, be sure to give the full path, starting with the first /. In our last assignment, suppose we wanted to add a link to always be able to get back to the home screen (root route). In our template, we would just need to add this:

    <!-- somewhere in our .Vue template section -->
    <router-link v-bind:to="{ path: '/' }">Home</router-link>

Which vue file should we put this in? There's not necessarily a right answer to this, but to try to be as DRY (don't repeat yourself) as possible, if we put this line in our App.vue file, we only have to write it once, and now it's available on every page! (Hopefully this is helping you to see the value of thinking in components!)

If we need to include variables in the path, we can use string concatenation or interpolation. Again, make sure you've got that leading /:

    <!-- with string concatenation -->
    <router-link v-bind:to="{ path: '/' + variableNameHere }">Some Section</router-link>

    <!-- with string interpolation (notice the back tick characters) -->
    <router-link v-bind:to="{ path: `/${variableNameHere}` }">Some Section</router-link>
