The DOM, a precursor

As a preface to the rest of this lesson, it's prescient to understand what the DOM is. DOM stands for "Document Object Model". Web developers throw this term around all the time, but it's important to understand that the DOM is really just the structure of the html represented as a JavaScript object. Each element has a series of properties and methods that give insight into anything you could ever want to know about the element and what is possible on the page. With that out of the way, let's move on to events.

DOM Events

DOM events give us the ability to wait and listen for something to happen on the page so we can react to it in our code. Some events are pretty self explanatory, like waiting for someone to click on an element. Others, we might not have thought about at first, like the text selection event. In fact, there are a ton of events we can use to our advantage. A full list can be found here. Below are some of the most common events we'll use in our daily lives:

  • click
  • keyup
  • mouseenter
  • mouseleave
  • dblclick
  • drag
  • drop

Vue Event Binding

Vue, like jQuery and other front-end libraries, makes it simple to listen for DOM events and react to them. Let's imagine that we have a button on our page and we want to print "clicked" in the console every time someone clicks the button. We'll do this using the v-on directive. v-on can be used to listen to any DOM event, but usually we'll be attaching events to a specific html element. For this example, we'll make a new vue app and put a button straight into the App.vue file.

App.vue

template
  <template>
    <button v-on:click="onClick()">Click Me!</button>
  </template>
script
  export default {
    name: 'app',
      methods: {
      onClick: function() {
        // eslint-disable-next-line
        console.log('clicked');
      }
    }
  }

Notice that the v-on directive is listening for the click event. Since the v-on directive is on the button tag, it will be triggered whenever the button is clicked. The v-on:click value specifies which method you'd like to run when the event gets triggered. Notice the onClick method in the component. Whenever someone clicks on the button, the onClick method will run! If you recreate this code and open up your web inspector you can see "clicked" print in the console! The // eslint-disable-next-line is to remove some default warnings that Vue will provide for us if we accidentally leave a console.log statement in our code.

This is a trivial example, but it truly is a fundamental building block for much more interesting applications. We'll see in the next assignment just how powerful this is.

A handy shortcut

Instead of using v-on:click, we can use a Vue provided shortcut. We can replace v-on with @. You can use @click instead of v-on:click. Just a heads up, you'll probably see @ more often in the real world, so keep an eye out!
