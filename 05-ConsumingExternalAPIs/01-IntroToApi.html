Intro to APIs

Objectives:

  • Understand what an API is

Sometimes when we are building an application, we may want to take advantage of existing data and/or tools rather than doing everything ourselves. Consider, for example, building an online store and using an external service to process payment information, or an event site that incorporates the weather, or a tool that integrates maps and directions. We could, conceivably, do all these things ourselves, but there are others in the community who specialize in these things specifically, so let's take advantage of that!

This means we'll have to leave our own application and make a call to an external server. An API, or application programming interface, is basically a given application's set of definitions that tell other applications how to access its services. Before integrating this immediately into our applications, let's first get some practice interacting with APIs. When working with web APIs, we use the HTTP request-response cycle to communicate with another server. Depending on the API, we'll need to consider a few different parts:

Request

  • method - indicates what type of action will be performed; most common HTTP verbs include GET, POST, PUT, PATCH, DELETE
  • headers - pass additional request information, commonly including authorization, authentication, or caching specifications
  • body - the specific data to send in the request

Response

  • status - codes indicate whether the request was successful or what went wrong; eg. 200 - OK, 404 - Not Found, 500 - Server Error (you can find a full list of status codes here: https://developer.mozilla.org/en-US/docs/Web/HTTP/Status)
  • content - the actual data returned from the request
    • In the request-response cycle generally, this is most commonly some combination of HTML, CSS, and JavaScript
    • When working with APIs, this is most commonly JSON data

To get a better understanding of how this works, we're going to use a tool called Postman. After getting comfortable with how APIs work, we'll drop Postman and integrate these calls directly into our applications, but Postman is a great tool during the development process to ensure the calls we intend to make are working as we expect.
