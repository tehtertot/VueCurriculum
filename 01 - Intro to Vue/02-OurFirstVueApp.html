Our First Vue Application

Objectives:

  • Get started using Vue
  • Learn about the Vue class

One of the nice things about Vue is how lightweight it is compared to other front-end frameworks. While we can spin up a server to host our web application, we can also just import it into our HTML file directly (much like adding the jQuery library to our HTML file).

Let's get started by creating a folder called hello_vue and add one new HTML file to it. Along with the standard head elements, add the following:

hello_vue/index.html

    <head>
        <!-- other meta tags -->
        <script src="https://cdn.jsdelivr.net/npm/vue/dist/vue.js"></script>
    </head>
    <body>
        <div id="app">
            Hello, {{ message }}
        </div>
    </body>

If you open this page in your browser, you'll just see: Hello, {{ message }}. In Vue (and in many templating engines), the double curly brackets indicate a variable. In our case, because we haven't defined any variables, it is simply rendered literally.

Let's go ahead and turn this into a Vue application! Create a file called index.js in the same folder. With the import statement added in the head tags, one of the things we now have access to is to a function called Vue, which takes an object as an argument. There are several different keys we can include in this object, but to start, we'll just focus on a few:

  • el - a stringe of the id of the element in scope, preceded by a # (same format as an id in css)
  • data - an object defining any variables being used, where the keys are the variable names

hello_vue/index.js

var app = new Vue({
    el: "#app",
    data: {
        message: "Vue!"
    }
})

That's a lot of curly braces, so take a second to break it down and understand the pieces. In the first line, we're creating a new Vue instance and passing in 1 argument. That argument is an object with 2 key-value pairs. The first key, el, indicates this instance is applicable to the HTML element with the id of "app". The second key, data, is an object that has one key-value pair. Since we used the variable name "message" in our HTML (remember the double curly braces?), we use that same name as the key in our data object.

Before we see this code in action, we'll need to link to this javascript file in our HTML.

hello_vue/index.html

    <!-- previous html... -->
    <body>
    <div id="app">
    Hello, {{ message }}
    </div>
        <!-- adding this line: -->
        <script src="index.js"></script>
    </body>

Why did we put it inside the body? We need to add it after we've imported the Vue framework, and also after we've defined the element we'll be applying the code to. Code is read from top to bottom, so go ahead and test that line in different places to see if you can get a couple of different errors! (Open your browser's console to see the errors.)
