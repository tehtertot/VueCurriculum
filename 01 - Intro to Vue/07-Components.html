If you took a peek inside the folder, you may be feeling a little overwhelmed. How did we go from 3 files to more than a dozen? We started you off with the basics because we wanted you to see fundamentally how Vue works. Now, we're going to change our mindset and approach Vue with a developer mindset.

Modularizing by Components

Now that we know we can do a lot with just a single HTML page, let's take a moment to think about working on a project with a team. Do you think it would be a good idea to all work on the same HTML file? How will you coordinate what you're working on to make sure it doesn't conflict with a teammate's work? What about that app variable? It will continue to grow as each teammate needs to add their own variables and other functionality. The assignments you've been working on so far have been small enough that you can complete them independently, but real world solutions will have a lot more moving parts, and this coordination problem will become significant.

With the rise of front-end frameworks, the notion of components has become the prominent approach to designing and working on the front end. Think of a component as a small subsection of a page. In Vue terms, components are reusable Vue instances with a name.

Take a look at the HelloWorld.vue file inside the src/components folder. This file defines a component named HelloWorld. Notice that there are three sections:

Template

Within the <template> tags is a snippet of HTML code. Notice there are no <head> or <html> sections. Instead, there is one and only one root element (in the example, the root element is the div with the class "hello"). The root element can have any number of child html elements, but each component can have only one root element, which is commonly a div.

Script

As you might expect, the script tags will hold the associated JavaScript code for a given component. As you can see from the pre-made template, now that we're using the Vue CLI to compile our code, the object being created is slightly different than the one we made previously:

    export default {
        name: 'HelloWorld',
        props: {
            msg: String
        }
    }
  • The export default line indicates that the following object will contain the information needed for Vue to create the component.
  • The name will be the name to be used when referencing this component. It's a good idea to make this the same name as the file, for consistency.
  • The props will be an object where the keys are any variable names you'll need in this component (similar to the data object defined in previous projects), but the values will be the data type. We'll learn where to define the actual values for these properties shortly.

Style

The style section is pretty straightforward. This is where we'll put the CSS we want to be applied to this component's HTML.

If you want the style to be scoped, or in other words, applied only to this component, be sure to include the scoped attribute in your style tag, like so: <style scoped>. For example, the HelloWorld component has the following styling:

    h3 {
        margin: 40px 0 0;
    }

Even though this is generically applied to any h3 tag, because it is scoped, it will only be applied to h3 elements in instances of the HelloWorld component. If you want to apply global styling (i.e. across all components in the entire application), we'll see in the next module where this should go.
