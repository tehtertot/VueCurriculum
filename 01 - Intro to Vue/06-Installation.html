Installation and App Creation

Objectives:

  • Learn about Vue CLI
  • Create a new Vue application using the Vue CLI

Now that you've experienced some of the basics of Vue, we're going to learn some of the developer tools that are available to us. We wanted you to see that, at the end of the day, the browser still only understands HTML, JavaScript, and CSS. As we learn these new tools, we'll be able to do things like:

  • have a more component-oriented mindset and write code accordingly,
  • easily add third-party packages (i.e. code written by others to enhance Vue's core capabilities),
  • run a server that compiles and auto-updates our code as we write

That might seem abstract at first, so let's just dive in.

Installation Steps

CLI stands for command-line interface. The Vue CLI is a specific interface that will allow us to create and run Vue applications via the command line. These steps explain how to install this tool.

  1. Check whether you have node installed by opening your command prompt or terminal and typing

    node -v

    If you see something like: v10.15.0, that tells you what version of Node you have installed.

    If you have a version lower than 8.11 or got an error like "'node' is not recognized..." or "command not found," you'll need to update/install node:

    1. Go to https://nodejs.org/en/ and download the latest version (the one recommended for most users). Follow the prompts and accept all the defaults.
    2. Once installation is complete, close any terminal or command prompt windows you have open and re-try step 1 to confirm it has been installed.

  2. Next, we'll install the Vue CLI by typing the following in the terminal:

    npm install -g @vue/cli

    npm install tells our system we want Node's package manager (npm) to install a package

    -g indicates we want that package installed globally onto our local machine (this means we only need to run this whole command one time)

    @vue/cli tells npm which package we want to install

Creating a New Vue Application

Now, every time we want to create a new Vue application, rather than manually creating and writing the HTML, CSS, and JavaScript files, the Vue CLI will create that and more for us:

  1. In the terminal, navigate to where the project should be saved.

  2. Type:

    vue create my-new-app

    Where my-new-app should be replaced with whatever you want to call your new app. (Note: the name of your app cannot include any capital letters.)

  3. When prompted to pick a preset, leave the arrow on default (babel, eslint) and hit enter. Once you feel more comfortable with Vue, you can start learning about the other options available.

    • What is Babel? Babel is a JavaScript compiler. As you'll see shortly, we'll be writing in more than just plain JavaScript, HTML, and CSS files, but that's the only thing our browser understands. Babel is the tool that will translate the Vue code we'll write into the languages understood by the browser.
    • What is ESLint? ESLint is a linter tool. A linter tool is a tool that identifies bugs and pattern warnings and reports them to you as the developer (i.e. those squiggly lines in your code indicating something might be wrong). ESLint is one specifically written for JavaScript. When we select this option, we are actually adding a version of ESLint on our project that understands the patterns of Vue.

  4. Once the app has been created (it may take a few minutes. You'll know it's done when you see the familiar prompt showing what directory you are in.) Type dir or ls to confirm a new folder has been created with the name specified in step 2.

  5. Navigate into the newly created folder and type

    npm run serve

  6. Once you see saying your project has been compiled and is running:

    Browse to http://localhost:8080 in your browser.

  7. Open the new folder in your text editor and take a look around. Can you find where the code displaying in your browser is coming from?

  8. In the HelloWorld.vue file, make a change to some of the text and save the file. Without refreshing the browser, do you see the changes? This is one of the features of the CLI serve function--it automatically detects changes to the Vue files and rebuilds! This makes development that much easier--be sure to keep your eye on the terminal in case of errors.
